# 常用排序算法说明

本文对应源码文件 `src/sort.cpp`，包含几种常见排序算法的实现。

## 1. 冒泡排序

### 原理
通过重复比较相邻两个元素，如果顺序错误就交换位置。每一轮都会把当前未排序区间中的最大值“冒泡”到末尾。

### 流程
1. 从头开始遍历数组，比较相邻元素。
2. 如果前一个元素大于后一个元素，则交换。
3. 一轮结束后，末尾元素有序。
4. 对剩余未排序部分重复以上步骤。
5. 如果某一轮没有发生交换，说明整体已经有序，可以提前结束。

### 复杂度
- 时间复杂度：最好 `O(n)`，平均/最坏 `O(n^2)`
- 空间复杂度：`O(1)`
- 稳定性：稳定

## 2. 选择排序

### 原理
每一轮从未排序区间中找出最小元素，放到当前应该放置的位置。

### 流程
1. 将数组分成已排序区和未排序区。
2. 在未排序区中找到最小值。
3. 将最小值与未排序区起始位置交换。
4. 已排序区向后扩展一位。
5. 重复直到全部有序。

### 复杂度
- 时间复杂度：最好/平均/最坏都为 `O(n^2)`
- 空间复杂度：`O(1)`
- 稳定性：不稳定

## 3. 插入排序

### 原理
把数组看成前半部分已经有序，每次取出一个新元素，插入到前面合适的位置。

### 流程
1. 默认第一个元素有序。
2. 从第二个元素开始，记录当前待插入元素。
3. 从已排序区间末尾向前比较。
4. 把比待插入元素大的值向后移动一位。
5. 找到合适位置后插入。

### 复杂度
- 时间复杂度：最好 `O(n)`，平均/最坏 `O(n^2)`
- 空间复杂度：`O(1)`
- 稳定性：稳定

## 4. 归并排序

### 原理
采用分治思想，先不断把数组拆成更小的子数组，等子数组天然有序后，再把两个有序子数组合并成一个更大的有序数组。

### 流程
1. 把当前区间从中间拆成左右两部分。
2. 递归地对左半部分排序。
3. 递归地对右半部分排序。
4. 将两个有序子数组按顺序合并。
5. 最终得到完整有序数组。

### 复杂度
- 时间复杂度：最好/平均/最坏都为 `O(n log n)`
- 空间复杂度：`O(n)`
- 稳定性：稳定

## 5. 快速排序

### 原理
选择一个基准值，把数组划分成“小于等于基准”和“大于基准”两部分，再分别递归排序。

### 流程
1. 选择一个基准元素。
2. 遍历区间，把小于等于基准的元素移动到左侧。
3. 将基准放到最终正确位置。
4. 对基准左边子区间递归排序。
5. 对基准右边子区间递归排序。

### 复杂度
- 时间复杂度：平均 `O(n log n)`，最坏 `O(n^2)`
- 空间复杂度：平均 `O(log n)`，最坏 `O(n)`（递归栈）
- 稳定性：不稳定

## 6. 堆排序

### 原理
先把数组构造成大顶堆，使堆顶元素始终为当前最大值。然后把堆顶与末尾交换，并重新调整堆，重复直到完成排序。

### 流程
1. 将数组构建为大顶堆。
2. 交换堆顶元素和当前末尾元素。
3. 缩小堆的有效范围。
4. 对新的堆顶执行下沉调整，恢复大顶堆性质。
5. 重复直到堆大小为 1。

### 复杂度
- 时间复杂度：最好/平均/最坏都为 `O(n log n)`
- 空间复杂度：`O(1)`
- 稳定性：不稳定

## 总结

如果数据规模较小，可以优先考虑冒泡、选择、插入这类简单排序。

如果更关注整体性能，归并排序、快速排序、堆排序更常用：
- 归并排序性能稳定，但需要额外空间。
- 快速排序平均性能优秀，但最坏情况较差。
- 堆排序空间占用低，但实现和常数因子通常比快速排序更重。
