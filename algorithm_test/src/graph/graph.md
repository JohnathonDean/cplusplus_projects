# 图论基础算法说明

本文对应源码文件 `src/graph/graph.cpp`，包含图论中几个最常见的基础算法实现：

- 广度优先搜索 `BFS`
- 深度优先搜索 `DFS`
- 无权图最短路径
- 拓扑排序
- 并查集 `Union Find`

## 1. 图的表示方式

这里使用`邻接表`表示图：

```cpp
using Graph = std::vector<std::vector<int>>;
```

含义是：
- 外层下标表示当前节点编号
- `graph[u]` 表示从节点 `u` 出发能够直接到达的所有邻接点

例如：

```cpp
Graph graph = {
    {1, 2},
    {3},
    {3},
    {}
};
```

表示：
- `0 -> 1`
- `0 -> 2`
- `1 -> 3`
- `2 -> 3`

## 2. 广度优先搜索 BFS

### 原理
从起点开始，一层一层向外扩展，先访问距离起点近的节点，再访问更远的节点。

### 核心数据结构
队列 `queue`

### 流程
1. 起点入队并标记已访问
2. 取出队头节点
3. 遍历它的所有邻接点
4. 将未访问过的邻接点入队
5. 重复直到队列为空

### 特点
- 适合求无权图最短路径
- 访问顺序体现“层次”

### 复杂度
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

## 3. 深度优先搜索 DFS

### 原理
从起点开始，沿一条路径不断向下搜索，直到不能继续为止，再回退到上一个节点继续搜索其他分支。

### 核心数据结构
- 递归调用栈
- 或显式栈 `stack`

### 流程
1. 访问当前节点并标记
2. 依次访问所有未访问的邻接点
3. 对邻接点递归执行相同过程
4. 当前分支走不通时回溯

### 适用场景
- 图遍历
- 连通性判断
- 环检测
- 回溯问题

### 复杂度
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

## 4. 无权图最短路径

### 原理
在无权图中，每条边的代价都相同，因此 BFS 第一次到达某个节点时，走过的边数一定最少。

### 流程
1. 起点距离设为 `0`
2. 起点入队
3. 每弹出一个节点，尝试更新它所有邻接点的距离
4. 若邻接点尚未访问，则距离设为 `当前节点距离 + 1`
5. 重复直到队列为空

### 结果
- 可达节点会得到最短边数
- 不可达节点保持为 `-1`

## 5. 拓扑排序

### 适用条件
仅适用于`有向无环图（DAG）`

### 原理
每次选取一个入度为 `0` 的节点加入结果，然后删除它发出的边，使新的入度为 `0` 的节点继续进入队列。

### 核心步骤
1. 统计所有节点的入度
2. 将所有入度为 `0` 的节点入队
3. 取出队头加入拓扑序
4. 删除它的出边，更新相邻节点入度
5. 新入度为 `0` 的节点继续入队

### 判环方法
如果最终加入结果的节点数少于图中总节点数，说明图中存在环，无法完成拓扑排序。

### 复杂度
- 时间复杂度：`O(V + E)`
- 空间复杂度：`O(V)`

## 6. 并查集 Union Find

### 作用
并查集用于高效维护“哪些节点属于同一个集合”。

它特别适合处理这类问题：
- 两个节点是否连通
- 动态合并两个集合
- 当前一共有多少个连通分量

### 核心操作

#### `find(x)`
查找节点 `x` 所在集合的根节点。

#### `unite(x, y)`
把 `x` 和 `y` 所在的两个集合合并起来。

#### `isConnected(x, y)`
判断两个节点是否属于同一个集合。

### 优化手段

#### 路径压缩
在查找根节点时，把沿途节点直接挂到根节点下面，减少后续查找成本。

#### 按大小合并
总是把较小的树挂到较大的树下面，避免树退化过高。

### 适用场景
- 无向图连通性判断
- 朋友圈/岛屿数量问题
- Kruskal 最小生成树
- 动态合并集合类问题

### 复杂度
在路径压缩和按大小合并同时使用时，单次操作的均摊复杂度非常接近 `O(1)`。

## 7. 总结

这几个算法是图论的基础模块：

- `BFS` 适合层次遍历和无权最短路
- `DFS` 适合深度探索和递归型问题
- 无权最短路本质上是 `BFS` 的直接应用
- 拓扑排序适合处理依赖关系和任务调度问题
- 并查集适合处理连通性和集合合并问题

如果后续继续扩展图论模块，比较自然的下一步是：
- Dijkstra
- 最小生成树
- Floyd
- A* 或 Bellman-Ford
